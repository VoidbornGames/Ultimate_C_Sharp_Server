<h1 id="contributing-to-ultimateserver">Contributing to UltimateServer</h1>
    <p>We welcome contributions to UltimateServer! This guide will help you get started. Whether you're fixing a bug, implementing a new feature, or improving documentation, your help is appreciated.</p>
    <h2 id="how-to-contribute">How to Contribute</h2>
    <h3 id="reporting-bugs">Reporting Bugs</h3>
    <p>If you find a bug, please report it by creating an issue on our <a href="https://github.com/VoidbornGames/Ultimate_C_Sharp_Server/issues">GitHub Issues</a> page.</p>
    <p>A good bug report includes:</p>
    <ul>
        <li>A clear and descriptive title.</li>
        <li>Steps to reproduce the behavior.</li>
        <li>The expected behavior.</li>
        <li>The actual behavior (including error messages, if any).</li>
        <li>Your environment details (OS, .NET version, etc.).</li>
    </ul>
    <h3 id="suggesting-features">Suggesting Features</h3>
    <p>Have an idea for a new feature? We'd love to hear it!</p>
    <ul>
        <li>First, check the existing issues to make sure it hasn't already been suggested.</li>
        <li>Create a new issue using the "Feature Request" template if available.</li>
        <li>Provide a clear description of the feature and why it would be useful.</li>
    </ul>
    <h3 id="submitting-changes-pull-requests">Submitting Changes (Pull Requests)</h3>
    <p>Follow these steps to submit your code changes:</p>
    <ol>
        <li><strong>Fork the repository</strong>: Click the "Fork" button on the top right of the project page.</li>
        <li><strong>Create a feature branch</strong>: In your forked repository, create a new branch for your change.
            <pre><code class="language-bash">git checkout -b feature/AmazingFeature</code></pre>
        </li>
        <li><strong>Make your changes</strong>: Implement your bug fix or new feature. Please follow our code style guidelines.</li>
        <li><strong>Commit your changes</strong>: Write a clear and concise commit message.
            <pre><code class="language-bash">git commit -m 'Add some AmazingFeature'</code></pre>
        </li>
        <li><strong>Push to the branch</strong>: Push your changes to your forked repository.
            <pre><code class="language-bash">git push origin feature/AmazingFeature</code></pre>
        </li>
        <li><strong>Open a Pull Request</strong>: Go to the original repository and click "New Pull Request". Select your branch and provide a detailed description of your changes.</li>
    </ol>
    <h2 id="development-setup">Development Setup</h2>
    <p>To set up a development environment:</p>
    <ol>
        <li>Follow the <a href="https://github.com/VoidbornGames/UltimateServer/wiki/Installation">Installation Guide</a> to get the server running.</li>
        <li>Clone your forked repository locally.</li>
        <li>Open the project in your preferred C# IDE (Visual Studio, VS Code, Rider).</li>
        <li>Ensure you have the .NET 8.0 SDK installed.</li>
    </ol>
    <h2 id="code-style">Code Style</h2>
    <p>We aim for clean, readable, and maintainable code. Please follow these general guidelines:</p>
    <ul>
        <li>Adhere to the Microsoft C# coding conventions.</li>
        <li>Use meaningful variable and method names.</li>
        <li>Add comments where the logic is complex or non-obvious.</li>
        <li>Keep methods and classes small and focused on a single responsibility.</li>
    </ul>
